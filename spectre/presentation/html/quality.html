Headline numbers

Populated automatically from each results/spectre/<run-id>/meta.json once you collect fresh runs with the new RunRecorder-instrumented spectre binary. See the How to regenerate section below for the exact commands.

1. Losslessness sanity check

Per-token log p_target(accepted) from spectre/metrics.csv (written by the softmax() instrumentation in spectre/src/main.cpp:266–288). 173 tokens, temperature 0.8. Bottom panel: cumulative perplexity. Re-running with --spec-type none produces an overlapping curve — empirical proof of losslessness.

2. Where is the difficulty hiding?

Distribution of p_target(x) on accepted tokens. The bimodal shape (mass near 1, long tail toward 0) is what an adaptive speculator should target: speculate aggressively on the easy mass, fall back on the hard tail.

3. Mean acceptance rate per speculator

From logs/benchmark-logs/*.log. Three runs per configuration. Note the gap between the single ngram_cache (0.355) and the chain (0.201) — chain pays a per-round overhead for running 5 speculators.

4. Efficiency frontier — speed vs. acceptance

The most counterintuitive result. Speculation is not free: below an empirical break-even of ≈ 0.10 acceptance, draft overhead exceeds savings and you end up slower than the autoregressive baseline.

5. Per-position acceptance — why long drafts saturate

Under an independent-Bernoulli model, the i-th drafted token is accepted with probability p̂ⁱ. p̂ is fitted from each speculator's global A/G ratio. The right panel shows expected accepted tokens per call vs. draft length k — saturates by k ≈ 6 even for the best speculator.

How to regenerate these figures

Two steps. First, run experiments — each spectre invocation produces one self-contained run directory: 

# target-only baseline
spectre/build/main --target-model <tgt.gguf> --seed 42 \
  --run-id ar-baseline-seed42 --prompt "..."

# speculative
spectre/build/main --target-model <tgt.gguf> --draft-model <dft.gguf> \
  --seed 42 --run-id spec-dft1-seed42 --prompt "..."

Then regenerate all figures (one command, reads every results/spectre/<run-id>/): 

python3 spectre/scripts/quality_eval.py

Each run directory contains:

  • meta.json — config snapshot + per-run aggregates + per-round summaries
  • tokens.csv — per-token observations (step,call,source,pos_in_draft,token_id,p_target,p_draft,logit,logprob)
