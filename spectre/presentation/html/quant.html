1. GGUF on-disk layout

GGUF is a self-contained, memory-mappable container: header + metadata KV map + tensor info table + raw tensor blob. llama.cpp mmaps the tensor data region directly into pages the matmul kernels read from - no copy, no deserialization step per inference.

Contrast with safetensors: safetensors only stores the raw tensors + a small JSON header (tensor names, dtypes, offsets). Everything else - tokenizer, config, chat template - lives in separate JSON files. GGUF folds all of that into one file, which is why a model is "one .gguf".

2. Conversion pipeline: HF → GGUF → quantized

Authors publish on HuggingFace as safetensors in BF16 or FP16. The path to a usable .gguf is two scripts that ship with llama.cpp.

3. Inside one Q4_K super-block (256 weights → 144 bytes)

"Block-wise scaling" is the central trick. Instead of one global scale per tensor (which gets blown up by outliers), Q4_K groups weights hierarchically and keeps per-group scales. The accountancy below explains where the "4.5 bits per weight effective" number for Q4_K_M comes from.

4. K-quant vs I-quant intuition

K-quants spend the same bit budget on every weight in a block; I-quants spend more bits on weights the calibration pass identifies as important. Same total bytes, different per-weight precision distribution.

5. Why quantize at all: memory bandwidth is the bottleneck

For one token from a 7B model on an A100: compute finishes in ~45 μs; moving the weights takes ~7 ms. The GPU sits idle 99% of the time waiting on memory. Quantization doesn't change the math; it shrinks the bytes that have to move.

6. Quality vs size Pareto curve

PPL Δ% on a held-out corpus vs bytes/weight. Above 4 bits the quality loss is within sampling noise; below 3 bits, reasoning chains start breaking; below 2 bits, only useful when "model doesn't load otherwise" is the alternative.

7. How quantization interacts with speculative decoding

Quantization and SD attack the same bottleneck (memory bandwidth) from different angles. They compose multiplicatively, which is the whole reason this thesis exists.

Self-speculative decoding (the Nemotron BF16 + Q8_0 pair in models/) collapses both ideas into one: the draft is the target, just at lower precision. The Q8 draft is ~50% faster to decode but produces nearly identical distributions, so acceptance is very high - quant choice is the speedup knob.
