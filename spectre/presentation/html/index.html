Code & data

The pieces that produced everything above.

  • spectre/src/main.cpp— custom libllama speculative decoder with structured-output instrumentation
  • results/spectre/<run-id>/— per-run meta.json + tokens.csv; one directory per spectre invocation
  • spectre/scripts/quality_eval.py— builds the 7 quality figures from results/spectre/*/
  • spectre/scripts/visualize.py— generic CSV plotter (older)
  • spectre/scripts/benchmark.sh— benchmark wrapper for the spectre CLI
  • benchmark.sh— llama-server benchmark across all --spec-type values (separate engine; logs in logs/)

Regenerate everything

Single commands to rebuild figures from source data.

# Sweep spectre over (mode × draft length × seed)
bash benchmark_spectre.sh

# Or a single run (writes results/spectre/<run-id>/{meta.json,tokens.csv})
spectre/build/main --target-model <tgt.gguf> [--draft-model <dft.gguf>] \
  --seed 42 --n-predict 256 --run-id my-run-id --prompt "..."

# 7 data-driven figures (results/spectre/*/ → quality-*.png)
python3 spectre/scripts/quality_eval.py

# Conceptual diagrams in atlas.html render client-side from Mermaid; no build step.

# Local server if file:// links misbehave
cd spectre/presentation/html && python3 -m http.server 8765
# then visit http://127.0.0.1:8765/
